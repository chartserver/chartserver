{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} SchemaHero is a Kubernetes Operator that will create and apply database schema migrations to any supported database, running in or out of the cluster. Instead of writing migrations, just edit the desired schema and deploy it. SchemaHero will compare it to the running schema, generate migration commands and apply them. {{% /blocks/lead %}} {{< blocks/section color="dark" >}} {{% blocks/feature icon="fa-lightbulb" title="Continuously rebased schema" url="/docs/concepts/declarative-schema-management/" %}} Instead of writing migrations, SchemaHero allows you to edit the desired schema, and let the migrations be calculated. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-gavel" title="Policy-enabled database schemas" url="/docs/advanced/enforcing-policies/" %}} Integrate and enable community tools such as Open Policy Agent to validate and enforce database schema practices. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="GitOps friendly" url="/docs/concepts" %}} YAML-based, easy-to-diff database schema definitions enable best practices for change management, review and gitops. {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section >}}
